# AD0-E136 Curriculum Map

Every lesson maps to an official exam sub-objective. Priority follows domain weight.
✅ = lesson exists · ⬜ = planned. Lesson numbers assigned when built.

## Domain 1 — Assessment (32%) — HIGHEST PRIORITY
- ⬜ Use the Best Practices Analyzer (BPA) — run, read reports, act on findings
- ⬜ Prepare & estimate a migration plan (phases, Cloud Acceleration Manager)
- ⬜ Assess which users, groups, permissions need to be moved
- ⬜ Review deprecated capabilities
- ⬜ Determine workflow requirements
- ⬜ Content-transfer requirements (Content Transfer Tool prereqs)
- ⬜ Asset-related implementation & impact (assessment lens)
- ⬜ Identify Dispatcher-related requirements

## Domain 2 — Design (16%)
- ⬜ Review existing components & validate vs latest version (Core Components)
- ⬜ Prepare configuration for asset microservices (processing profiles)
- ⬜ Caching strategy (CDN, Dispatcher, TTL, invalidation)

## Domain 3 — Deployment (16%)
- ⬜ Configure Cloud Manager pipelines & tie to environments
- ⬜ Provision AEMaaCS environments (dev/stage/prod × author/publish/preview)
- ⬜ Manage deployment with Adobe I/O CLI (aio)

## Domain 4 — Migration (20%)
- ⬜ Execute tests (quality gates, functional/UI tests)
- ⬜ Asset migration strategy
- ⬜ Base/initial content migration strategy
- ⬜ Execute migration tools: Content Transfer Tool, Modernization, Refactoring — report & refine
- ⬜ Plan post go-live support

## Domain 5 — Implementation (16%)
- ✅ 0001 — Mutable vs immutable repo split (project structure foundation)
- ⬜ Merge multiple projects into one pipeline-deployable project
- ⬜ RepoInit scripts for users/groups/permissions
- ⬜ Set up local SDK (AEMaaCS SDK, local quickstart)
- ⬜ Update configs based on run modes available on AEMaaCS

## Cross-cutting
