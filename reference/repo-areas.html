AEMaaCS Repository Areas

Which paths are mutable, which are immutable, and where each ships from. Keep this open while restructuring a project. See Lesson 1.

PathTypeShips inNotes
/appsImmutableui.apps (application pkg)Code. Read-only at runtime. Baked into golden image.
/libsImmutableAdobe (product)Never modify. Product code.
/contentMutableui.contentAuthored pages & assets. Changed at runtime.
/confMutableui.contentEditable templates, policies, context-aware config.
/homeMutableRepoInit (structure)Users, groups, service users. Only publish path writable.
/varMutableruntimeRuntime data. Do not deploy content here.
/etcMutableui.content (legacy)Deprecated — migrate off /etc where possible.
/oak:indexMutableui.apps or ui.content*Index defs ship with project but repo area is mutable; must be refactored for cloud.
Package type rules Application package (<packageType>application</packageType>): immutable paths only.
Content package (<packageType>content</packageType>): mutable paths only.
Container package: embeds both; produced by the all module.
Deploy mantra Code → immutable → pipeline only. Content → mutable → author + Package Manager / distribution. Never the two in one package.

Source: Adobe Experience League — AEM Project Content Package Structure.
